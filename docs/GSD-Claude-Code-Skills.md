# GSD — Claude Code Skills Reference

This document explains the five Claude Code skills installed in this repository and how to use them when working on GSD or on projects generated by the GSD pipeline.

---

## What Are Claude Code Skills?

Claude Code skills are Markdown files with YAML frontmatter stored in `.claude/skills/<name>/SKILL.md`. Claude Code reads the skills index in `CLAUDE.md` and applies the matching skill automatically when you describe a task that fits. You can also invoke any skill explicitly using its slash command.

**All skill files are in**: `.claude/skills/`

## Skill 1 — SQL Expert

**File**: `.claude/skills/sql-expert/SKILL.md`
**Command**: `/sql-expert`

### When to Activate

- Designing a new table or schema for a generated project
- Writing or modifying a stored procedure
- Normalizing a data model from a spec
- Writing CTEs, recursive queries, or window functions
- Generating idempotent migration scripts
- Modeling a new relationship or upsert pattern

### Example Prompts

```
/sql-expert design a Notifications table with multi-tenant isolation for the ChatAI project

/sql-expert write usp_Notification_Create with audit columns and tenant scoping

/sql-expert generate a migration script for the RoleModules table, idempotent
```

### What It Enforces

- **SP-only rule**: All DB access via stored procedures — no inline SQL
- **Naming**: `usp_{Entity}_{Action}` SPs, `IX_{Table}_{Columns}` indexes, `UQ_`/`CK_`/`FK_` constraints
- **Audit columns**: `CreatedAt`, `CreatedBy`, `UpdatedAt`, `UpdatedBy`, `IsDeleted` on every table
- **Multi-tenant**: `TenantId NVARCHAR(50) NOT NULL` + `WHERE TenantId = @TenantId AND IsDeleted = 0`
- **Idempotent migrations**: `IF NOT EXISTS` wrapped — safe to re-run
- **Error handling**: `BEGIN TRY / THROW / END CATCH` always present
- **Grants**: `GRANT EXECUTE ON ... TO [AppRole]` after every SP

## Skill 2 — SQL Performance Optimizer

**File**: `.claude/skills/sql-performance-optimizer/SKILL.md`
**Command**: `/sql-performance-optimizer`

### When to Activate

- A generated stored procedure is running slowly
- You need to understand an execution plan from a project's DB layer
- Adding or tuning indexes after profiling
- Diagnosing N+1 query patterns in generated repository code
- CPU or IO is high on a specific query in a running project

### Example Prompts

```
/sql-performance-optimizer this SP is slow — what indexes should I add?

/sql-performance-optimizer explain what's wrong with this correlated subquery

/sql-performance-optimizer what execution plan operators should I watch for on large tenant tables?
```

### What It Provides

- **Workflow**: Read query → identify costly operators → diagnose bottleneck → prescribe fix
- **Operator guide**: Table Scan, Key Lookup, Hash Match, Sort, Nested Loops — cost and fix for each
- **Index design**: Covering index pattern (equality → range → INCLUDE), filtered indexes for soft-delete
- **Query rewrites**: Correlated subquery → aggregated JOIN; `COUNT(*)` → `EXISTS`; parameter sniffing fix
- **Blocking**: `sys.dm_exec_requests` queries, RCSI recommendation
- **SP checklist**: `SET NOCOUNT ON`, explicit lengths, no `SELECT *`, temp tables for large result sets

## Skill 3 — React UI Design Patterns

**File**: `.claude/skills/react-ui-design-patterns/SKILL.md`
**Command**: `/react-ui-design-patterns`

### When to Activate

- Reviewing generated frontend screens for missing async states
- Adding loading skeletons to an existing component
- Implementing error handling with retry on a screen
- Adding optimistic delete/create/update with React Query
- Designing a confirmation dialog for a destructive action
- Adding an error boundary to a route or component tree

### Example Prompts

```
/react-ui-design-patterns review the ConnectorsPage — does it handle all five states?

/react-ui-design-patterns implement optimistic delete on the assistants list

/react-ui-design-patterns add a loading skeleton to match the UserCard layout
```

### The Five States Rule (mandatory for all generated screens)

| State | Trigger | Fluent UI Pattern |
|---|---|---|
| Loading | `isLoading === true` | `<Skeleton>` matching final layout |
| Error | `isError === true` | `<MessageBar intent="error">` + retry |
| Empty | `data` is `[]` or `null` | Centered EmptyState + CTA |
| Populated | data has items | Normal render |
| Optimistic | mutation pending | Disabled UI + React Query `onMutate` cache update |

## Skill 4 — Composition Patterns

**File**: `.claude/skills/composition-patterns/SKILL.md`
**Command**: `/composition-patterns`

### When to Activate

- A generated component has accumulated boolean props
- Designing a reusable UI primitive for a project
- Deciding whether to use context vs props vs compound components
- Reviewing component architecture in a generated frontend
- Working with shared state between sibling components in a form or wizard

### Example Prompts

```
/composition-patterns refactor this Panel component — it has 7 boolean props

/composition-patterns design a compound component API for the AssistantCard

/composition-patterns should I use context or prop drilling for this wizard flow?
```

### Core Rules

| Rule | Summary |
|---|---|
| `architecture-avoid-boolean-props` | Use composition and explicit variants — no boolean mode switches |
| `architecture-compound-components` | Group related UI as `Card`, `Card.Header`, `Card.Body`, `Card.Footer` |
| `state-context-interface` | Typed `{ state, actions, meta }` on context — never expose raw `setState` |
| `patterns-explicit-variants` | Named variant components over `<Alert type="error" />` |
| `patterns-children-over-render-props` | `children` composition over `renderHeader` / `renderFooter` props |

## Skill 5 — Web Design Guidelines

**File**: `.claude/skills/web-design-guidelines/SKILL.md`
**Command**: `/web-design-guidelines`

### When to Activate

- Auditing a generated screen for accessibility compliance
- Before submitting a PR that touches frontend files
- Checking ARIA usage, keyboard navigation, or color contrast
- Verifying Fluent UI v9 components are used accessibly in generated code

### Example Prompts

```
/web-design-guidelines src/Client/technijian-spa/src/pages/ConnectorsPage.tsx

/web-design-guidelines review the AppShell component for accessibility

/web-design-guidelines src/Client/technijian-spa/src/components/**/*.tsx
```

### How It Works

1. Fetches live guidelines from `vercel-labs/web-interface-guidelines` (always latest)
2. Reads the specified files
3. Checks against all rules
4. Reports findings in `file:line` format

### Fluent UI v9 ARIA Checklist (Built-In)

| Rule | Check |
|---|---|
| ARIA-01 | All interactive elements have `aria-label` or visible `<Label>` from Fluent |
| ARIA-02 | `role` attributes match element semantics |
| ARIA-03 | Focus order follows visual order — no `tabIndex > 0` |
| ARIA-04 | Color contrast ≥ 4.5:1 for text, ≥ 3:1 for UI components |
| ARIA-05 | All form inputs paired with `<Label>` from `@fluentui/react-components` |
| ARIA-06 | Dialog/Drawer has `aria-labelledby` pointing to title |
| ARIA-07 | Toast/Alert uses `role="alert"` for errors, `role="status"` for info |
| ARIA-08 | Images have meaningful `alt` text (`alt=""` for decorative) |
| ARIA-09 | All actions reachable via Tab + Enter/Space |
| ARIA-10 | No keyboard traps outside Modal/Dialog |

## Using Skills in Practice

### Auto-Activation

Describe the task naturally — Claude detects the matching skill:

> "Write a stored procedure for the notification settings table with tenant isolation"
> → SQL Expert activates

> "This query on ChatMessages is taking 3 seconds"
> → SQL Performance Optimizer activates

> "Add a loading skeleton and error state to the WorkflowRunsPage"
> → React UI Design Patterns activates

### Explicit Invocation

Force a skill with the slash command, useful when combining with a specific task:

```
/sql-expert [task description]
/sql-performance-optimizer [task description]
/react-ui-design-patterns [task description]
/composition-patterns [task description]
/web-design-guidelines [file or pattern]
```

### Combining Skills

Chain skills for multi-layer work. Example — adding a new feature end-to-end:
1. `/sql-expert` — design the table + SP
2. `/sql-performance-optimizer` — validate index strategy
3. `/react-ui-design-patterns` — implement the React screen with all five states
4. `/web-design-guidelines` — audit the new screen for accessibility

## Adding New Skills

To add a skill:

1. Create `.claude/skills/<skill-name>/SKILL.md` with YAML frontmatter (`name`, `description`, `metadata`)
2. Write the skill body: activation criteria, patterns, code examples
3. Add a reference entry under `## Installed Skills` in `CLAUDE.md`
4. Add an entry in the skills table at the top of this doc

Skills can be sourced from [mcpmarket.com](https://mcpmarket.com) or authored from scratch. Keep each skill focused on one domain.
